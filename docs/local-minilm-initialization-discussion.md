# 本地 MiniLM 初始化时机问题讨论

> 日期：2026-07-17
> 状态：问题讨论；本地单用户前提已确定，初始化流程尚未实施
> 范围：本地 Embedding 模型的下载、缓存与交互时机，不调整现有项目计划版本

## 1. 背景

项目默认使用 Chroma 的本地 `all-MiniLM-L6-v2` ONNX 模型生成文本向量。原因是当前默认聊天模型接口不提供 Embedding 能力，而混合检索 RAG 需要向量表示论文片段和用户问题。

该模型首次使用时需要下载约 79.3 MB。若下载由某个普通业务请求隐式触发，用户只能看到请求长时间未完成，很容易误判为模型 API、论文检索或系统卡死。

本讨论以以下前提为基础：

- 项目首先服务于本机、单用户场景；
- 前后端和向量数据均运行、保存于本地；
- 暂不处理多用户部署、集中式模型缓存和云端模型分发；
- MiniLM 下载失败不应影响在线发现论文等无关功能。

## 2. MiniLM 在项目中的作用

| 功能 | 是否依赖 MiniLM | 说明 |
|------|----------------|------|
| Semantic Scholar / arXiv 在线找论文 | 否 | 使用论文源自身的检索与排序 |
| BM25 关键词检索 | 否 | 只依赖分词和文本统计 |
| 论文片段向量化 | 是 | 导入后建立向量索引 |
| 语义向量检索 | 是 | 用问题向量匹配论文片段向量 |
| BM25 + 向量混合 RAG | 是 | MiniLM 提供稠密检索部分 |
| LLM 回答生成 | 否 | 由前端或 `.env` 配置的聊天模型完成 |

结论：MiniLM 是完整 RAG 链路的重要本地依赖，但不是在线论文检索和聊天模型本身的依赖。

## 3. 已观察到的问题

一次 `Text-to-SQL` 在线检索暴露了以下串行等待：

1. Semantic Scholar 返回 `429`；
2. 系统切换到 arXiv 并取得候选论文；
3. 候选论文重排首次创建本地 Embedding 客户端；
4. Chroma 隐式开始下载约 79.3 MB 的 MiniLM 模型；
5. 前端仍显示“正在检索论文”，用户无法判断实际正在下载模型。

在线找论文并不要求本地向量模型，因此在这个阶段下载 MiniLM 属于职责错位。

## 4. 当前代码行为

当前已完成以下隔离：

- 在线论文检索会先检查 MiniLM 是否已经完整缓存；
- 模型未缓存时，不触发下载，直接采用 Semantic Scholar 或 arXiv 的相关性排序；
- 模型已缓存时，才使用标题和摘要向量做本地候选重排；
- 前端会显示检索词、当前论文源、候选数量和等待时间。

当前尚未解决：第一次导入论文并建立向量索引时，仍可能由 Chroma 隐式触发 MiniLM 下载。

## 5. 下载时机方案

### 方案 A：后端启动时自动下载

优点：后续导入和检索无需等待。

问题：后端启动会被网络速度阻塞；下载失败容易表现为服务无法启动；与本地开发的快速启动目标冲突。

结论：不推荐。

### 方案 B：第一次导入论文时自动下载

优点：无需增加独立操作，只有真正使用向量索引时才下载。

问题：模型下载、PDF 处理和索引构建混在同一个进度中；失败原因不清晰；慢速网络下仍像导入卡死。

结论：可作为兜底，但不应静默执行。

### 方案 C：环境安装后显式初始化

在安装 Python 依赖后、第一次导入论文前，运行独立命令，例如：

```bash
python -m app.index.prepare_embeddings
```

优点：下载行为和失败原因清晰；可单独重试；不阻塞后端启动和在线找论文。

问题：多一个初始化步骤，需要 README 和状态检查配合。

结论：当前本地项目的推荐方案。

### 方案 D：前端提供“初始化向量模型”操作

前端启动后检查缓存状态，在用户开始导入前显示模型大小、下载按钮和进度。

优点：对非开发用户最直观。

问题：需要增加状态接口、后台下载任务、进度协议和失败重试，超出当前小功能范围。

结论：作为后续体验增强，不在本次实现。

## 6. 当前建议

本地项目采用“显式初始化为主、首次导入兜底提示”的策略：

1. 后端启动时只检查缓存，不自动下载；
2. 在线找论文永远不触发 MiniLM 下载；
3. 环境安装完成后，通过独立命令初始化 MiniLM；
4. 第一次导入前再次检查缓存；若未就绪，应明确提示需要初始化，而不是静默等待；
5. 模型下载完成后复用 Chroma 本地缓存；
6. 若配置 `EMBEDDING_PROVIDER=openai_compatible`，则不要求本地 MiniLM。

## 7. 建议验收标准

- 未安装 MiniLM 时，在线论文检索可以正常完成；
- 后端启动不触发模型下载；
- 初始化命令明确显示目标模型、下载大小、进度和缓存位置；
- 下载中断后可安全重试，并由 Chroma 校验模型文件；
- MiniLM 未就绪时，论文导入返回清晰错误或进入明确的初始化流程；
- 初始化完成后，论文导入、向量检索和混合 RAG 正常工作；
- 不在仓库中提交模型文件或下载临时文件。
